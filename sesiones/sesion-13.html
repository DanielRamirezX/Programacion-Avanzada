    V · Errores comunes — NO / SÍ

    El parcial no castiga lo difícil.
    Castiga lo que se hace por costumbre mal.

    Estos cuatro son los que más se repiten en el Bloque I. A la izquierda, lo que NO; a la derecha, lo que SÍ. Reconócelos hoy para no escribirlos el viernes:

    ✗ Así NO
    class Cuenta: # atributo de CLASE, no de objeto saldo = 0 def depositar(monto): saldo += monto # ¿quién es saldo?
    Falta self. Sin él, no hay método de instancia ni atributo propio del objeto.
    edad = "18" if edad >= 18: # TypeError print("mayor")
    Comparar texto con número. "18" es str, no int. Revienta.
    nums = [10, 20, 30] print(nums[3]) # IndexError
    Índice fuera de rango. 3 elementos = índices 0,1,2. El último NO es 3.
    if activo = True: # SyntaxError ...
    Asignar en vez de comparar. = asigna; == compara.
    ✓ Así SÍ
    class Cuenta: def __init__(self): self._saldo = 0 def depositar(self, monto): self._saldo += monto
    Atributo por objeto, encapsulado con _, y métodos que reciben self.
    edad = int("18") if edad >= 18: print("mayor")
    Convierte primero. int(...) pasa el texto a número antes de comparar.
    nums = [10, 20, 30] print(nums[-1]) # 30, el último for n in nums: # sin índices print(n)
    Índice negativo o for-in. -1 es el último; recorrer evita el error.
    if activo == True: ... if activo: # aún mejor ...
    Compara con == — y si es booleano, if activo: basta.

    VI · Q&A — trae tus dudas

    Hoy es tu turno de preguntar.
    La pregunta tonta es la que no se hace.

    Repaso colaborativo en parejas: junta tus 5 minutos de dudas con tu compañero y traigan las que no resuelvan entre los dos. Para arrancar, estas son las que cada ciclo aparecen — si alguna te suena, ya sabes qué levantar la mano a preguntar:

    cuál es la diferencia exacta entre la clase y el objeto?
    El molde vs lo que sale del molde. Una clase, muchos objetos.
    cuándo uso for y cuándo while?
    ¿Sabes cuántas veces repetir? for. ¿Repites hasta que algo pase? while.
    por qué mi atributo lleva guion bajo si igual puedo tocarlo?
    Encapsulamiento por convención: el _ dice "no me toques desde fuera".
    qué pasa si recorro una lista y la modifico al mismo tiempo?
    Cosas raras. Recorre una copia o construye una lista nueva.
    La IA como compañero de repaso — úsala bien
    • ✓  Pídele que te haga preguntas: "hazme 5 preguntas de opción múltiple sobre encapsulamiento en Python".

    • ✗  Pedirle "resuélveme el examen" — en el parcial estás tú solo. Si la IA lo entendió por ti, tú no lo entendiste.
