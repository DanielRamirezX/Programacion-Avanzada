La fábrica
de software.

Tu primer encargo real: una empresa ecológica necesita un sistema de inventario y ventas. Hoy no eres alumno — eres la fábrica de software que lo va a construir.

I · El encargo

Una empresa real (bueno, casi)
te contrata para resolver un problema real.

Practicario MCT158 · Práctica 1 de 6 · texto íntegro del caso

Eres parte de una fábrica de software que se le encarga desarrollar un sistema para gestionar los productos y ventas de una empresa ecológica. La empresa busca diferenciarse en el mercado al ofrecer un inventario eficiente para sus sucursales y quiere optimizar la experiencia del usuario mediante el registro digital de ventas.

La fábrica de software construirá al sistema digital con una orientación a objetos. Entre sus aspectos conceptuales realizará el análisis, diseño y abstracción. Con lo anterior, se podrán establecer las clases con atributos y métodos, así como los objetos y mensajes entre ellos.

Se realiza una entrevista entre el gerente de la empresa, los usuarios del sistema, el ingeniero en requerimientos y el desarrollador. En esta junta se establecen como válidos los requerimientos que debe cumplir el sistema: permitir agregar nuevos productos al inventario, registrar ventas incluyendo el cálculo del total, y generar reportes con la lista de productos y las ventas realizadas. El desarrollo del programa será en Java.

Requerimiento 01
Agregar productos
Permitir registrar nuevos productos al inventario de la empresa.
Requerimiento 02
Registrar ventas
Capturar ventas incluyendo el cálculo del total de cada una.
Requerimiento 03
Generar reportes
Mostrar la lista de productos y las ventas realizadas.

II · Análisis — la técnica del subrayado

Las clases se esconden en los sustantivos.
Los métodos, en los verbos.

Antes de escribir código, un desarrollador lee el caso como detective: los sustantivos que se repiten son candidatos a clases o atributos; los verbos de acción son candidatos a métodos. Practica la técnica con este caso de otro dominio — toca cada palabra subrayada:

Un refugio de animales registra adopciones de sus mascotas. Cada mascota tiene nombre, especie y cuota de adopción. El refugio calcula el donativo total recaudado y muestra la lista de mascotas disponibles.
Sustantivo principal → clase
Sustantivo descriptivo → atributo
Verbo de acción → método
Toca las palabras subrayadas para clasificarlas. Llevas 0 de 8.

Ahora aplica la misma técnica al caso de la empresa ecológica. Lee el encargo de arriba con lápiz en mano. Estas preguntas te guían — las respuestas las encuentras tú:

Qué "cosas" maneja el sistema? Esos son tus candidatos a clase.
Pista: en el texto del encargo hay al menos 3.
Qué datos describe cada cosa? (un producto tiene…, una venta tiene…)
Esos son los atributos — y decide cuáles deben ser privados.
Qué acciones pide cada requerimiento? Agregar, registrar, calcular, mostrar…
Cada verbo es un método. ¿En qué clase vive cada uno?
Cómo se relacionan? ¿Una venta contiene productos? ¿El inventario contiene…?
Las relaciones definen las flechas de tu diagrama de clases.

VI · El plan de ataque

No empieces por el código.
Nunca empieces por el código.

Orden recomendado de trabajo
  1. Análisis (papel y lápiz): subraya el caso, identifica clases, atributos, métodos y relaciones.
  2. Diagrama de clases: dibújalo ANTES de programar. Si el diagrama está mal, el código nacerá mal.
  3. Código en Java: una clase a la vez. Compila seguido — no escribas 100 líneas sin probar.
  4. Los 3 requerimientos: agregar productos → venta con total → reporte. En ese orden, probando cada uno.
  5. Diagrama de secuencia: ahora que el código funciona, dibuja cómo se hablan los objetos.
  6. El documento: arma portada, introducción, pega código y evidencias, responde conclusiones, cita bibliografía.
  7. Sube también el proyecto a tu repo de GitHub — la práctica es institucional, pero tu portafolio es tuyo.
