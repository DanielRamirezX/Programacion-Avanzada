I · Qué es una API REST

Tu programa deja de ser una isla.
Se vuelve un servicio que otros consumen.

Una API (Application Programming Interface) es un contrato: defines qué pueden pedirte y qué les devuelves. REST es el estilo más usado para APIs web: organiza todo alrededor de recursos (productos, usuarios, ventas), cada uno con una URL, y se opera con los verbos del protocolo HTTP. Tu base de datos queda detrás; la API es el mostrador de atención.

Concepto 01
Recurso
Una "cosa" que tu sistema maneja, expuesta con una URL: /productos, /productos/7. Lo que en POO era una clase.
Concepto 02
Verbo HTTP
La acción sobre el recurso: GET leer, POST crear, PUT actualizar, DELETE borrar. El verbo es la intención.
Concepto 03
JSON
El idioma común: datos como texto estructurado. La API recibe JSON y responde JSON, no importa quién la llame.

La gran promesa de REST: quien consume tu API no necesita saber Python. Una app de React, un script en Go, una hoja de cálculo o un teléfono Android pueden llamarla igual. Tú expones el contrato; ellos hablan HTTP.

II · Los verbos del protocolo HTTP

Cuatro verbos.
Cuatro intenciones distintas.

El mismo recurso /productos cambia de significado según el verbo. Aprende a leerlos: ¿es idempotente (repetirlo da el mismo resultado)? ¿lleva body (datos en el cuerpo de la petición)? Toca cada verbo:

GET /productos
POST /productos
PUT /productos/7
DELETE /productos/7

III · Los códigos de respuesta

Cada respuesta trae un número.
Ese número ya cuenta la historia.

Antes de leer el JSON, el código de estado HTTP te dice qué pasó. Los de la familia 2xx son éxito; los 4xx son culpa de quien pidió (mandó mal los datos); los 5xx son culpa del servidor (algo se rompió de tu lado). Toca cada código:

200
OK
201
Created
404
Not Found
422
Unprocessable
500
Server Error
2xx · éxito 4xx · error del cliente 5xx · error del servidor

IV · FastAPI — del método al endpoint

Tu método de Python,
con un decorador, se vuelve endpoint.

FastAPI es un framework de Python para construir APIs: rápido, moderno y con validación automática gracias a Pydantic. La idea clave: una función Python normal se convierte en una ruta de tu API solo con poner un decorador encima. Lo que devuelvas con return se serializa a JSON solo. Compara una API mínima completa:

main.py — la API
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

# el modelo: forma del dato
class Producto(BaseModel):
  nombre: str
  precio: float

# una ruta = decorador + función
@app.get("/productos")
def listar():
  return productos
cómo se lee

app = FastAPI() crea tu aplicación: el mostrador completo.

@app.get("/productos") dice: "cuando llegue un GET a esta URL, ejecuta la función de abajo".

La función listar() es Python puro — el que ya sabes. Lo que devuelve se vuelve JSON solo.

Lo arrancas con uvicorn main:app --reload y queda escuchando.

@app.post("/productos")  # crear: recibe un Producto en el body
def crear(p: Producto):
  productos.append(p)
  return p  # FastAPI responde 201 / 200 con el JSON

VI · Validación automática con Pydantic

No escribes ni un if de validación.
El modelo lo hace por ti.

Aquí está el superpoder de FastAPI. Declaras la forma del dato con un modelo Pydantic y, si el body que llega no cumple, FastAPI rechaza la petición solo con un 422 — antes de que tu función corra. Manda dos bodies al mismo endpoint POST /productos y mira la diferencia:

el modelo que exige FastAPI
class Producto(BaseModel):
  nombre: str
  precio: float
  stock: int
