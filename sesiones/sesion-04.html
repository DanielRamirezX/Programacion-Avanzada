III · El constructor invisible

Aunque no escribas __init__,
Python siempre pone uno.

Y por defecto, no hace nada. Pulsa para revelar el constructor invisible:

Al crear el objeto
ana = Estudiante()
objeto: ana
(sin atributos — el __init__ default no hace nada)
→ El objeto existe pero está vacío.

V · El guardián __name__

El mismo archivo, dos comportamientos.
Depende de cómo lo llames.

Cuando ejecutas un archivo directo, __name__ vale "__main__" y el código corre. Cuando lo importas, vale el nombre del módulo y el código se salta. Cambia el escenario:

$ python estudiante.py
__name__ = "__main__"
✓
Puerta abierta — el código corre.
$ python main.py → importa estudiante
__name__ = "estudiante"
✗
Puerta cerrada — el código se salta.
class Estudiante: def __init__(self, nombre): self.nombre = nombre if __name__ == "__main__": ana = Estudiante("Ana") print(ana.nombre) # solo si se ejecuta directo

VI · Lo que toda clase tiene gratis

Acabas de ver __name__ a nivel módulo.
Cada clase trae el suyo — y muchos más.

Python te regala una colección de atributos y métodos en toda clase. Algunos pertenecen a la clase, otros a cada instancia. Todos heredados de object. Toca cada uno y mira qué devuelve:

a nivel clase · sobre Estudiante
a nivel instancia · sobre ana
Hay muchos más — los __add__, __len__, __iter__, __call__ y compañía aparecen cuando quieres que tu clase se comporte como número, lista, iterable o función. Los verás cuando los necesites; por ahora basta con saber que existen.
